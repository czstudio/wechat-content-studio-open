# End-to-End Example

## User Ask

```text
帮我写一篇公众号，主题是 ReAct 框架的局限性。我已经有一些素材，今天想直接开写，后面还要发前 review。
```

## Route

1. Topic Board
2. Draft Brief
3. Draft
4. Proof Gate

## Winner Snapshot

- Working Title: 为什么很多 Agent 团队最后都会卡在 ReAct 这一步
- Opening Scene: 演示里每一步都像对的，最后整体却跑偏
- Write Readiness: Ready now

## Draft Brief Snapshot

- Promise: 讲清楚 ReAct 为什么容易局部正确、整体失真，以及更稳的替代路径怎么选
- Spine:
  - ReAct 的局限不在“不会想”，而在“不会先定边界”
  - 三种替代范式的根本区别
  - 怎么按任务结构选方案

## Proof Gate Snapshot

| Slot | Status | Note |
|---|---|---|
| Opening Scene | Present | 有失败感 |
| Middle Proof | Present | 有框架对比 |
| Ending Lift | Weak | 结尾判断不够收束 |
